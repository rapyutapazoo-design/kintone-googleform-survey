# Kintone × Google Form アンケートシステム セットアップ手順書

フォルダ監視型・常設GAS 1本の構成です。Kintoneボタンと掲示QRコードの**2導線**で回答を集約し、
**部屋番号＋氏名で個人ごとに名寄せ**して重複回答を防ぎます（Kintoneで回答済みの人はQRからもブロックされます）。
一度セットアップすれば、以後の運用でコード・ID・URLに触れる作業は発生しません。

> **前提:** Kintoneユーザーの**従業員ID（employeeNumber）に部屋番号が登録**されていること（自動入力に使用）。

> **表記について:** `<サブドメイン>` や `<アンケート管理アプリID>` のように `< >` で囲んだ箇所は、
> 自環境の値に読み替えてください。自環境の値の控えには `docs/env.local.example.md` を
> コピーした `docs/env.local.md`（Git管理外）の利用を推奨します。

| ファイル | 配置先 |
|---|---|
| `gas/survey-gas.js` | Google Apps Script（常設プロジェクト1本） |

## ステップ1: Google Drive の準備

1. Google Driveに監視用フォルダ「**アンケート**」を作成
2. フォルダIDを控える（URLの `folders/` の後ろの文字列。`?` 以降は含めない）
3. **マスターフォーム**を作成する（⚠️ フォルダの**外**に置くこと）
   - フォーム名: `[マスター] アンケートひな型` （先頭の `[マスター]` は登録除外の目印。フォルダ内に誤って入っても登録されない）
   - 設定タブで以下を設定（コピー作成時に引き継がれる）:

| 設定 | 値 |
|---|---|
| メールアドレスを収集 | **収集しない**（GASが自動でオフにするが、マスター側もオフ推奨） |
| ログインを必須にする | **オフ** |
| 送信後に回答を編集 | **オフ** |
| 説明文 | 「回答は1人1回です。送信後の修正はできません」等を記載 |
| 確認メッセージ | 「控えメールが見当たらない場合は迷惑メールフォルダをご確認ください」を追記推奨 |

> **回答の控えメールについて:** Form標準の「回答のコピーを送信」は使いません。
> GASがフォーム生成時に「メールアドレス（控えの送信先・任意）」質問を自動追加し、
> ボタン経由で開くと**Kintoneユーザー情報の登録アドレスが自動入力**されます。
> 回答時にアドレスが入力されていれば、GASが控えメールを送信します（空欄なら送信なし）。
> 送信クォータはGASアカウントの上限（無料: 100通/日、Workspace: 1,500通/日）に従います。
> 差出人はGASを実行しているGoogleアカウントになります。

## ステップ2: アンケート管理アプリのフィールド設定

### 必要なフィールド構成

| フィールド名 | フィールドコード | 種類 | 備考 |
|---|---|---|---|
| アンケートタイトル | `survey_title` | 文字列(1行) | GASが自動入力（フォーム名を転記） |
| GoogleフォームURL | `form_url_base` | 文字列(1行) | GASが自動入力（事前入力URL） |
| フォームID | `form_id` | 文字列(1行) | GASの照合キー |
| エントリーID | `entry_id` | 文字列(1行) | 参考情報（GASが自動入力） |
| ステータス | `status` | ドロップダウン | 選択肢: `作成中` / `受付中` / `終了` |
| 投票ボタンスペース | `vote_button_space` | スペース | 要素IDを `vote_button_space` に設定 |

**回答記録テーブル**（フィールドコード: `answer_log`）:

| サブフィールド | コード | 種類 |
|---|---|---|
| 部屋番号 | `ans_room` | 文字列(1行) |
| 氏名 | `ans_name` | 文字列(1行) |
| 回答経路 | `ans_route` | 文字列(1行) | （Kintone / QR） |
| 回答日時 | `ans_time` | 日時 |

### アクセス権

- 一般ユーザー: レコード**閲覧のみ**（編集・追加・削除は不可）
- 管理者（`survey-portal.js` の `ADMIN_ORG_CODE` に設定した組織）: フル権限
- `answer_log` には氏名（個人情報）が入るため、一般ユーザーには portal.js で非表示にする

### ユーザー情報の前提（重要）

- Kintoneユーザーの**従業員ID欄に部屋番号を登録**しておくこと。`portal.js` が `getLoginUser().employeeNumber` を部屋番号として自動入力する。未登録ユーザーは部屋番号が空欄になる

### APIトークン

- 権限: **レコード閲覧 ✅ + 編集 ✅ + 追加 ✅**
- 発行後、**アプリを「更新」**すること（更新しないとトークンが無効）

### 一覧の準備

- PC用一覧: 「アンケートタイトル」「GoogleフォームURL」「ステータス」を表示（URL列がボタンに置換される）
- モバイル用一覧: 名前を `スマホ版簡易一覧` で作成（`survey-portal.js` の `MOBILE_VIEW_NAME` と一致させる）

## ステップ3: 常設GASのセットアップ

1. [Google Apps Script](https://script.google.com/) で新規プロジェクトを作成（名前: `アンケート自動化（常設）` 等）
2. `gas/survey-gas.js` の内容を `コード.gs` に貼り付け
3. **スクリプトプロパティ**を設定（プロジェクトの設定 → スクリプトプロパティ）:

| プロパティ名 | 値 |
|---|---|
| `SURVEY_APP_ID` | `<アンケート管理アプリID>` |
| `API_TOKEN` | ステップ2で発行したトークン |
| `SUBDOMAIN` | `<サブドメイン>`（`https://〇〇.cybozu.com` の 〇〇 部分） |
| `FORM_FOLDER_ID` | ステップ1のフォルダID |

4. `testConfig` を手動実行して設定値を確認（初回は権限承認ダイアログが出る。メール送信・Drive・外部接続の承認が必要）
5. **時間トリガーを設置**: トリガー → 追加 → 関数 `watchFolder` / イベント: 時間主導型 / **5分おき**
6. **ウェブアプリとしてデプロイ**（QR中継ページ `doGet` のために必須）:
   - デプロイ → 新しいデプロイ → 種類「ウェブアプリ」
   - 実行ユーザー: **自分** ／ アクセスできるユーザー: **全員**
   - デプロイ後の**ウェブアプリURL**を控える（QRコード生成に使用）
   - `testConfig` を再実行すると `webAppUrl=...` がログに出る（控え忘れ時の確認用）

> ⚠️ Webhookは使いません。ウェブアプリURLはコード更新で原則変わりませんが、
> デプロイをやり直すとURLが変わる場合があり、その際はQRの再生成が必要です。

## ステップ4: portal.js の適用

1. `kintone/survey-portal.js` 冒頭の「★設定」ブロックを自環境の値に書き換える
   - `ADMIN_ORG_CODE`: 管理者組織のコード（**必須**）
   - `SPACE_ID` / `MOBILE_VIEW_NAME`: 既定値から変えた場合のみ
2. アンケート管理アプリ → 設定 → JavaScript / CSSでカスタマイズ
3. 書き換え済みの `survey-portal.js` をアップロード（**PC用・モバイル用の両方**）
4. アプリを更新

## ステップ5: QRコードの作成（アンケートごと）

掲示用QRは、**フォームのURLではなくGAS中継ページのURL**を指す必要があります（回答済みブロックのため）。

- QRに埋め込むURL: `<ウェブアプリURL>?fid=<フォームID>`
  - `<ウェブアプリURL>` はステップ3-6で控えたURL
  - `<フォームID>` は対象アンケートの `form_id`（Kintoneレコードで確認、またはフォーム編集URLの `/d/●●/edit` の ●●）
- 任意のQR作成ツールでこのURLをQR化し、掲示物に印刷する
- 掲示物に「お一人様1回」「部屋番号とお名前の入力が必要です」等を添える

## ステップ6: 動作確認（エンドツーエンド）

1. マスターフォームを右クリック → 「コピーを作成」→「アンケート」フォルダへ移動 → 名前（`[マスター]` を外す）と質問を編集
2. **最大5分待つ** → アンケート管理アプリにレコードが自動作成されていることを確認
   - `survey_title` / `form_url_base` / `form_id` / `entry_id` が入っている
   - フォームに「部屋番号」「氏名」「回答経路（変更しないでください）」「メールアドレス（控えの送信先・任意）」質問が追加されている
3. ステータスを `受付中` に変更して保存
4. **【Kintone導線】** 一般ユーザーでログイン → ボタンから回答 → **部屋番号・氏名・メールが自動入力**されていることを確認 → 送信 → 再読み込みで「✅ 回答済み」、`answer_log` に経路=Kintoneで記録
5. **【QR導線・未回答】** 別人の部屋番号・氏名でQR（中継ページURL）を開く → 部屋番号・氏名入力 → フォームへ進める → 回答 → `answer_log` に経路=QRで記録
6. **【QR導線・ブロック】** 手順4で回答済みの部屋番号・氏名でQRを開く → 「✅ すでに回答済みです」が表示されフォームに進めないことを確認（要件4）
7. **【表記ゆれ】** 手順4の氏名を全角／半角・空白違いでQR入力 → 同一人物と判定されブロックされることを確認
8. 控えメールが届くこと（空欄なら送信なし・記録は正常）を確認
9. （補修テスト）`answer_log` から行を削除 → 5分以内に自動復元されることを確認
10. ステータスを `終了` に変更 → 5分以内にフォーム受付停止。QRを開くと「受付を終了しました」、Kintoneの未回答者は「⛔ 受付終了」表示

## 【毎回】管理者の運用フロー

```
① マスターフォームを「コピーを作成」→「アンケート」フォルダへ → 質問編集
② 最大5分待つ（自動でKintoneに登録される）
③ アンケート管理アプリでステータスを「受付中」に → 公開
　 （期間中の作業なし。回答状況は answer_log、内容はFormの回答タブ）
④ 終了時はステータスを「終了」に変更するだけ
　 （5分以内にフォーム受付も自動停止）
```

> **アンケートを完全に取りやめる場合:** 先にステータスを「終了」にして5分待ち
> （送信トリガーの自動掃除を済ませてから）、フォームを削除してください。
> ゴミ箱内のフォームは監視対象外のため、再登録されることはありません。

## トラブルシューティング

| 症状 | 確認ポイント |
|---|---|
| レコードが自動作成されない | GASの「実行数」で `watchFolder` のログ確認。フォームがフォルダ内にあるか、`[マスター]` 接頭辞が付いていないか |
| `[401]` エラー | APIトークンの値と権限（閲覧+編集+追加）。トークン発行後にアプリを更新したか |
| 回答済みにならない | 部屋番号・氏名が変更されていないか。同一人物の氏名表記が大きく異なると別人扱いになる。5分待って補修同期でも反映されないかを確認 |
| 部屋番号が自動入力されない | Kintoneユーザー情報の**従業員ID**に部屋番号が登録されているか（未登録なら空欄） |
| QRを開くとエラーになる | QRのURLが `<ウェブアプリURL>?fid=<form_id>` 形式か。GASがウェブアプリとしてデプロイ（アクセス: 全員）済みか |
| QRで回答済みブロックが効かない | QR入力の氏名表記が登録時とずれていないか。`answer_log` の `ans_room`/`ans_name` を確認 |
| 控えメールが届かない | 迷惑メールフォルダ確認。フォームのメール欄が空欄でなかったか。GASの「実行数」で `onFormSubmit` のメール送信エラー有無を確認 |
| ボタンが表示されない | portal.jsがPC用・モバイル用両方に適用されているか。スペース要素ID `vote_button_space` |
| スマホでカードが出ない | `MOBILE_VIEW_NAME` と同名の一覧が存在するか |
| 同時開催が増えてトリガー上限に達した | GASの送信トリガーは約20個が上限。終了済みアンケートのステータスを「終了」にすればトリガーは自動削除される |
