# Kintone × Google Forms アンケートシステム

Kintoneを「アンケートの入り口」、Google Formを「アンケート本体」として組み合わせ、
両者の接続を**常設のGoogle Apps Script（GAS）1本**で全自動化するシステムです。

管理者はGoogle Driveの指定フォルダにフォームを作るだけ。ID・URLのコピペ、
スクリプトの編集、Webhookやトリガーの設定作業は一切不要です。

## 特徴

- **2導線で回答を集約** — Kintoneボタン（職員/登録ユーザー）と掲示QRコード（部屋番号・氏名を入力する住民）の両方を1フォームで管理
- **個人単位の重複防止** — 部屋番号＋氏名で名寄せ。**Kintoneで回答済みの人はQRからもブロック**（中継ページで「回答済み」表示）
- **コピペゼロ運用** — フォルダ監視GASがフォームを検知し、Kintoneへの登録・トリガー設置まで自動実行
- **回答済みの即時反映** — 送信トリガーで回答者を記録し、Kintone上のボタンが「✅ 回答済み」に切り替わる（5分おきの補修同期付きで自己修復）
- **回答経路の記録** — Kintone/QRの内訳を `answer_log` に記録し集計で区別できる
- **回答データはGoogleに集約** — Kintoneには回答内容を保存せず、回答の確認・集計はFormの回答タブで完結
- **控えメールの自動送信** — 控え送信先のメールアドレスはKintoneユーザー情報から**自動入力**（任意・空欄なら送信なし）。回答時にGASが控えメールを送信
- **受付終了も自動** — Kintoneのステータスを「終了」にするだけで、フォームの受付停止・QR受付終了表示・不要トリガーの掃除まで自動

## アーキテクチャ

```
┌─ Google Drive「アンケート」フォルダ ─────────────┐
│  マスターフォーム（フォルダ外・コピー元）             │
│  各アンケートフォーム（管理者がコピー作成）            │
└──────────┬───────────────────────┘
           │ 監視（5分おき・時間トリガー1個）
┌─ 常設GAS 1本 ─────────────────────────┐
│ ① 新フォーム検知 → 初期設定 + トリガー設置          │
│    + Kintoneへレコード自動作成                     │
│ ② 回答送信時 → 回答者コードを即時記録               │
│    （5分おきの差分同期で取りこぼしを自動補修）        │
│ ③ ステータス「終了」検知 → フォーム受付の自動停止     │
└──────────┬───────────────────────┘
           │ REST API（APIトークン）
┌─ Kintone アンケート管理アプリ ─────────────────┐
│ 一覧・詳細・モバイルカードにボタンを表示              │
│ 📝 回答する ／ ✅ 回答済み ／ ⛔ 受付終了            │
└──────────────────────────────────┘
```

## 運用フロー（毎回）

```
① マスターフォームを「コピーを作成」→ 監視フォルダへ → 質問編集
② 最大5分待つ（自動でKintoneに登録される）
③ Kintoneでステータスを「受付中」に変更 → 公開
④ 終了時はステータスを「終了」に変更するだけ
```

## 動作要件

- Kintone（cybozu.com）環境とアプリ管理権限
- Google アカウント（Drive / Forms / Apps Script）
- 回答者はGoogleアカウント不要（フォームはログイン不要設定で運用）

## 制約・既知の事項

- GASのフォーム送信トリガーはプロジェクトあたり約20個が上限のため、同時開催アンケート数に上限があります（終了したアンケートのトリガーは自動削除されます）
- 個人単位の名寄せは「部屋番号＋氏名」で行います。氏名は手入力（QR）のため、表記ゆれ（全半角・空白）は正規化しますが、大きく異なる表記は別人と判定され重複を許す可能性があります
- QR中継ページのため**GASをウェブアプリとしてデプロイ**（アクセス: 全員）する必要があります
- 自己申告ベースのため、他人の部屋番号・氏名を騙る不正までは防げません（性善説）
- 控えメールはGASが送信するため、送信クォータ（無料: 100通/日、Workspace: 1,500通/日）に従います
